## 1. ขอบเขตของนโยบาย

นโยบายนี้อธิบายการประมวลผลข้อมูลเมื่อคุณใช้แอป PostDee และ API ของ PostDee เพื่อเลือกวิดีโอ สร้างหรือแก้ไขข้อความด้วย AI เชื่อมบัญชีโซเชียล โพสต์หรือตั้งเวลา ดูผลการโพสต์ รับการแจ้งเตือน จัดการแพ็กเกจ และลบบัญชี

PostDee พึ่งพาผู้ให้บริการภายนอกหลายรายตามหัวข้อ 6 ผู้ให้บริการเหล่านั้นอาจเป็นผู้ควบคุมหรือผู้ประมวลผลข้อมูลแยกต่างหากตามบริการของตน

## 2. ข้อมูลที่ระบบประมวลผล

| ประเภทข้อมูล | ตัวอย่างที่พบในระบบปัจจุบัน |
| --- | --- |
| บัญชีและการยืนยันตัวตน | Firebase UID, อีเมล, ชื่อที่แสดง, URL รูปโปรไฟล์, สถานะยืนยันอีเมล และสถานะยืนยันหมายเลขโทรศัพท์ หมายเลขโทรศัพท์และการส่ง OTP จัดการผ่าน Firebase Authentication |
| วิดีโอและเนื้อหา | ไฟล์วิดีโอ เสียงหรือเฟรมที่คัดจากวิดีโอ ชื่อไฟล์ ประเภทไฟล์ ขนาดไฟล์ แคปชั่น แฮชแท็ก เทมเพลต ข้อความประกอบ และตัวเลือกการตัดต่อ |
| ข้อมูลการโพสต์ | แพลตฟอร์มที่เลือก วันเวลาโพสต์ สถานะคิวและการเผยแพร่ รหัสโพสต์จากแพลตฟอร์มภายนอก ผลสำเร็จ/ล้มเหลว และข้อความผิดพลาด |
| การเชื่อมต่อโซเชียล | ชื่อแพลตฟอร์ม รหัส integration ของ PostPeer ชื่อบัญชีที่แสดง รหัสบัญชีภายนอก และ PostPeer profile ID ระบบปัจจุบันไม่เก็บรหัสผ่านของบัญชีโซเชียลในฐานข้อมูล PostDee |
| แพ็กเกจและการซื้อ | แพ็กเกจ สถานะสมาชิก วันสิ้นสุดรอบ รหัสลูกค้าหรือรหัสสมาชิก และเหตุการณ์วงจรสมาชิกที่ RevenueCat, App Store หรือ Google Play ส่งให้ระบบ PostDee ไม่ได้รับข้อมูลบัตรชำระเงินโดยตรงจาก flow ปัจจุบัน |
| โควตาและสถิติ | จำนวนหน่วยโพสต์ จำนวนครั้งสร้างแคปชั่น AI จำนวนนาทีตัดต่อ AI ยอดวิว ยอดถูกใจ และสถิติรวมตามแพลตฟอร์มหรือช่วงเวลา |
| การแจ้งเตือนและอุปกรณ์ | Firebase Cloud Messaging token และชนิดแพลตฟอร์มของอุปกรณ์ เมื่อผู้ใช้อนุญาตการแจ้งเตือนและระบบ Firebase Messaging ทำงาน |
| ข้อมูลการใช้งานและข้อผิดพลาด | หมวดเหตุการณ์ เช่น เริ่ม/สำเร็จ/ล้มเหลวในการเข้าสู่ระบบ การเลือกวิดีโอ และผลการโพสต์ รวมถึง exception และ stack trace ผ่าน Firebase Analytics และ Firebase Crashlytics เมื่อ Firebase เปิดใช้งาน โค้ด event ที่กำหนดเองไม่ส่งเนื้อหาวิดีโอ แคปชั่น หรืออีเมลเป็น parameter |
| ข้อมูลทางเทคนิคและความปลอดภัย | IP address ที่ API ใช้เป็นกุญแจจำกัดอัตราคำขอ และข้อมูลคำขอที่จำเป็นต่อการตรวจสอบ token การป้องกันการใช้งานเกิน และการแก้ปัญหา |
| ข้อมูลที่เก็บเฉพาะในอุปกรณ์ | ธีม สถานะผ่าน onboarding แบบร่างโปรไฟล์ แบบร่าง Link in Bio และค่าร่างของเครื่องมือที่ยังอยู่ระหว่างพัฒนา ผ่าน SharedPreferences |

วิดีโอหรือเสียงที่คุณเลือกอาจมีใบหน้า เสียง ข้อมูลสินค้า หรือข้อมูลของบุคคลอื่น คุณต้องมีสิทธิและฐานที่เหมาะสมก่อนอัปโหลดหรือเผยแพร่ข้อมูลเหล่านั้น

## 3. แหล่งที่มาของข้อมูล

เราได้รับข้อมูลจาก:

1. คุณ เมื่อเข้าสู่ระบบ กรอกข้อมูล เลือกวิดีโอ สร้างโพสต์ ตั้งเวลา หรือเลือกการตั้งค่า
2. Firebase Authentication เมื่อยืนยันตัวตน อีเมล หรือหมายเลขโทรศัพท์
3. PostPeer และแพลตฟอร์มโซเชียลที่คุณเลือกเชื่อมต่อ
4. RevenueCat, Apple App Store และ Google Play สำหรับสถานะการซื้อและสมาชิก
5. SDK และระบบของ PostDee สำหรับเหตุการณ์การใช้งาน ข้อผิดพลาด โควตา และความปลอดภัย

## 4. วัตถุประสงค์ในการใช้ข้อมูล

เราใช้ข้อมูลเพื่อ:

- สร้างและรักษา session ของบัญชี ตรวจสอบสิทธิแพ็กเกจ และป้องกันการเข้าถึงโดยไม่ได้รับอนุญาต
- อัปโหลด ประมวลผล พรีวิว และส่งวิดีโอไปยังแพลตฟอร์มที่ผู้ใช้เลือก
- จัดคิว ตั้งเวลา ติดตามผล และแจ้งผลการโพสต์
- สร้างแคปชั่น ถอดเสียง และจัดทำคำแนะนำการตัดต่อด้วย AI
- แสดงเทมเพลต ปฏิทิน โควตา และสถิติของบัญชี
- จัดการการซื้อ การกู้คืนสิทธิ และเหตุการณ์ต่ออายุ/ยกเลิกแพ็กเกจ
- วิเคราะห์ความเสถียรของแอป ตรวจจับข้อผิดพลาด จำกัดอัตราคำขอ และรักษาความปลอดภัย
- ดำเนินการตามคำขอลบบัญชีและข้อกำหนดที่ใช้บังคับ

## 5. การประมวลผลด้วย AI

เมื่อคุณเลือกใช้ฟีเจอร์ AI ระบบอาจส่งสื่อที่เกี่ยวข้องไปยังผู้ให้บริการที่ตั้งค่าไว้ดังนี้:

- **Google Gemini**: ใช้วิดีโอหรือเสียงจากคลิป และสำหรับบางแพ็กเกจอาจใช้เฟรมที่คัดจากคลิป เพื่อช่วยสร้างแคปชั่น SEO แฮชแท็ก หรือ hook
- **Groq**: ใช้สื่อหรือเสียงจากคลิปสำหรับการถอดเสียงด้วย Whisper และสร้างแผนการตัดต่อ

ผลลัพธ์ AI อาจไม่ถูกต้อง ไม่ครบถ้วน หรือไม่เหมาะกับบริบท ผู้ใช้ควรตรวจสอบก่อนตัดต่อหรือเผยแพร่ทุกครั้ง

## 6. ผู้ให้บริการและผู้รับข้อมูล

ระบบปัจจุบันอาจส่งข้อมูลเท่าที่จำเป็นให้ผู้ให้บริการต่อไปนี้:

| ผู้ให้บริการ | หน้าที่ใน PostDee | ข้อมูลที่เกี่ยวข้อง |
| --- | --- | --- |
| Google Firebase | Authentication, Analytics, Crashlytics และ Cloud Messaging | บัญชี/การยืนยันตัวตน เหตุการณ์การใช้งาน ข้อมูลข้อผิดพลาด และ device token |
| Render และ PostgreSQL บน Render | โฮสต์ API และฐานข้อมูล production ใน region Singapore ตาม `render.yaml` | บัญชี โพสต์ เทมเพลต สมาชิก โควตา device token และรหัสการเชื่อมต่อ |
| Cloudflare R2 | เก็บวิดีโออัปโหลดชั่วคราวและให้ signed URL | ไฟล์วิดีโอและ metadata การอัปโหลด |
| Google Gemini | สร้างแคปชั่นและคำแนะนำจากคลิป | วิดีโอ/เสียง เฟรม และคำสั่งที่เกี่ยวข้อง |
| Groq | ถอดเสียงและวางแผนตัดต่อ AI | สื่อ/เสียงและคำสั่งที่เกี่ยวข้อง |
| PostPeer | เชื่อมบัญชีและเผยแพร่ไปยังแพลตฟอร์มโซเชียล | รหัสบัญชี/integration แคปชั่น แพลตฟอร์ม เวลาโพสต์ และ signed URL ของวิดีโอ |
| TikTok, YouTube, Instagram และ Facebook หรือแพลตฟอร์มที่ผู้ใช้เลือก | รับและเผยแพร่โพสต์ | เนื้อหา วิดีโอ แคปชั่น และข้อมูลที่ผู้ใช้สั่งเผยแพร่ |
| RevenueCat, Apple App Store และ Google Play | ซื้อ กู้คืน และซิงก์สิทธิสมาชิก | รหัสผู้ใช้/สมาชิก ผลิตภัณฑ์ สถานะและเหตุการณ์การซื้อ |

เราไม่พบ flow สำหรับขายข้อมูลส่วนบุคคลให้บุคคลอื่นในระบบปัจจุบัน และ PostDee จะไม่ขายข้อมูลส่วนบุคคลโดยไม่ได้แก้ไขนโยบายและดำเนินการตามกฎหมายที่เกี่ยวข้อง

เราอาจเปิดเผยข้อมูลเมื่อจำเป็นเพื่อปฏิบัติตามกฎหมาย คำสั่งที่ชอบด้วยกฎหมาย ป้องกันความเสียหาย หรือบังคับใช้ [ข้อกำหนดการใช้บริการ](./TERMS_OF_SERVICE.md)

## 8. ระยะเวลาการเก็บและการลบข้อมูล

- ข้อมูลบัญชี โพสต์ เทมเพลต การเชื่อมต่อ สมาชิก และโควตาถูกเก็บในฐานข้อมูลตราบเท่าที่บัญชียังใช้งานหรือเท่าที่จำเป็นต่อบริการ
- วิดีโออยู่ใน Cloudflare R2 เพื่อรองรับการอัปโหลด AI การโพสต์ทันทีหรือที่ตั้งเวลา และการดึงไฟล์โดย PostPeer การลบหลังโพสต์สำเร็จเป็น best-effort และขึ้นกับการตั้งค่า production
- signed upload URL ปัจจุบันมีอายุประมาณ 5 นาที และ managed multipart session มีอายุประมาณ 1 ชั่วโมง ระยะเวลาดังกล่าวเป็นอายุ URL/session ไม่ใช่คำรับรองว่าไฟล์วิดีโอถูกลบภายในเวลานั้น
- Firebase Analytics, Crashlytics, Firebase Authentication, RevenueCat และผู้ให้บริการภายนอกเก็บข้อมูลตามการตั้งค่าบัญชีและข้อกำหนดของตน
- แบบร่างและค่าตั้งที่อยู่ใน SharedPreferences จะอยู่บนอุปกรณ์จนกว่าจะถูกเขียนทับ ล้างข้อมูลแอป หรือลบแอป Flow ลบบัญชีปัจจุบันล้าง session เข้าสู่ระบบ แต่ยังไม่ได้ล้างแบบร่างในเครื่องทุกประเภทโดยอัตโนมัติ

เมื่อผู้ใช้กด **ลบบัญชี** และการลบสำเร็จ ระบบจะยกเลิกงานที่อยู่ในคิว ตัดการเชื่อมต่อ PostPeer ลบวิดีโอใต้ owner prefix ลบข้อมูลของผู้ใช้ในฐานข้อมูล และลบ Firebase identity เป็นลำดับสุดท้าย การลบบัญชี **ไม่ยกเลิกสมาชิกใน App Store หรือ Google Play** ผู้ใช้ต้องจัดการสมาชิกในร้านค้าก่อนหรือแยกต่างหาก

## 9. ความปลอดภัย

มาตรการที่มีในระบบรวมถึง Firebase ID token verification, การแยกข้อมูลตามผู้ใช้, signed URL อายุสั้น, server-side secret, owner-scoped media key, HTTP security headers, rate limiting และขั้นตอนลบบัญชีที่ป้องกันงานใหม่ระหว่างการลบ

อย่างไรก็ตาม ไม่มีระบบใดรับประกันความปลอดภัยได้ทั้งหมด หากสงสัยว่าบัญชีถูกเข้าถึงโดยไม่ได้รับอนุญาต โปรดติดต่อ `support@postdee.app` **[TODO: ยืนยันช่องทางและกระบวนการรับแจ้งเหตุข้อมูลรั่วไหล]**

## 10. สิทธิและตัวเลือกของคุณ

- เลือกว่าจะให้แอปเข้าถึงคลังรูปภาพเพื่อเลือกวิดีโอหรือไม่
- เลือกว่าจะอนุญาตการแจ้งเตือนหรือไม่ โดยแอปขอ permission เมื่อผู้ใช้กดเปิดการแจ้งเตือน
- แก้ไขข้อมูลร่างโปรไฟล์ในแอป และตัดการเชื่อมต่อบัญชีโซเชียลที่รองรับ
- ลบบัญชีถาวรจากหน้าโปรไฟล์ โดยอาจต้องเข้าสู่ระบบใหม่หาก session เก่าเกินกำหนด
- ล้างแบบร่างเฉพาะเครื่องด้วยการล้างข้อมูลแอปหรือลบแอป

เมื่อ Firebase ถูกเปิดใช้งาน โค้ดปัจจุบันเปิด Firebase Analytics และ Crashlytics อัตโนมัติและยังไม่มีสวิตช์ opt-out แยกในแอป
